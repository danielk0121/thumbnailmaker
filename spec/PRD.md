# Product Requirements Document (PRD)

## 1. 프로젝트 목적
별도의 프레임워크 없이 순수 웹 기술(HTML, CSS, JS)과 SVG를 활용하여 빠르고 가볍게 구동되는 텍스트 기반 썸네일 생성 도구 제작 (웹 브라우저 및 로컬 `file://` 환경 모두 완벽 지원)

## 2. 핵심 기능 요구사항

### 텍스트 편집
- **입력 방식**: 4줄 높이의 텍스트 영역(textarea) 제공 (기본 문구: "썸네일\n글자를\n입력해주세요")
- **자동 줄바꿈(Word Wrap)**: 한글 및 영문 텍스트가 SVG 가로 너비(90%)를 넘어가면 자동으로 줄바꿈 처리
- **수직 정렬**: 줄 수가 늘어나거나 글자 크기가 변해도 항상 썸네일 중앙에 정렬 유지
- **크기 조절**: 숫자 직접 입력 및 밀착된 확대(+), 축소(-) 버튼을 통한 10px 단위 세밀한 크기 조절 (최소 10px, 최대 500px)

### 디자인 설정
- **배경 모드 (버튼 그룹 선택)**:
  - **단색(Solid)**: 배경색 1만 적용. 배경색 2 영역은 흑백/반투명 처리되어 직관적인 비활성화 상태 제공
  - **그라데이션(Gradient)**: 배경색 1과 2를 혼합한 그라데이션 적용
- **색상 지정 및 랜덤**: 각 색상(1, 2)별 컬러 피커 제공 및 개별 '랜덤 색상' 버튼 지원
- **모서리 스타일 (버튼 그룹 선택)**:
  - **각지게(Sharp)**: 모서리 둥글기 0px
  - **둥글게(Round)**: 모서리 둥글기 40px 고정 적용

### 해상도 및 레이아웃
- **해상도 설정**: 기본값 400x400. 가로/세로 수치 입력 후 '적용' 버튼 클릭 시 썸네일 크기 변경
- **실제 크기 미리보기**: 썸네일은 설정된 실제 픽셀 크기로 렌더링되며, 큰 해상도 시 컨테이너 내에서 스크롤 가능
- **반응형 지원 (Mobile/Desktop)**:
  - 썸네일 이미지는 화면 너비를 넘어가지 않도록 자동 축소(`max-width: 100%`)
  - 1000x1000 데스크톱 화면에서 한눈에 들어오는 콤팩트한 여백 및 컨트롤 패널 높이 적용
  - 340px 모바일 화면에서도 깨짐 없는 해상도 입력 영역 및 레이아웃 제공

### 파일 추출 및 저장
- **포맷 지원**: PNG, JPG, SVG 다운로드 버튼 제공
- **안정성 강화**:
  - `file://` 환경에서의 보안 제약을 우회하기 위한 Base64 Data URI 기반의 인코딩 처리
  - SVG를 Canvas로 변환 시 폰트 정보 유지를 위한 인라인 스타일 동적 주입 및 Canvas 오염 방지 로직 적용
- **사용자 경험**:
  - 저장 파일명 포맷: `thumbnail-YYYYMMDD-HHmmss.[확장자]` 적용
  - 다운로드 성공 시 브라우저 알림(alert) 제공

## 3. 기술 스택 (Technical Spec)
- **Language**: JavaScript (Vanilla JS)
- **Markup**: HTML5, SVG (Scalable Vector Graphics)
- **Styling**: Vanilla CSS (CSS3)
- **Directory**: `app/` 폴더 내 모든 소스 코드 관리
- **Deployment**: GitHub Pages (`app/index.html` 직접 공개)
- **SEO/OG Tags**: `app/assets/og-image.png`를 활용한 Open Graph 썸네일 및 파비콘 구성

## 4. 정책 및 컨벤션
- **언어**: UI 및 문서 전반에서 이모지를 배제한 깔끔한 텍스트 톤 유지. 모든 커밋 메시지와 주석은 한글 사용.
- **HTML 가독성**: `textarea` 내부 텍스트와 같이 들여쓰기 문제가 발생하는 데이터는 JS 단에서 초기화 처리.
